编程语言分类

编程语言分类1. 按代际分类2. 按编译方式分类3. 按照数据类型4. 按照程序设计4. 其他分类

笔者常常看见很多关于编程语言的术语，见得多了，虽说大概能猜测出一二。但还是觉得不妥，于是乎便整理如下。

1. 按代际分类

  • 第一代：机器语言 cpu指令集 0,1组成
  • 第二代：汇编语言
  • 第三代：高级语言 java/c/c++
  • 第四代：应用语言 sql

2. 按编译方式分类

  • 编译型

    • 一次编译到二进制码，然后运行
    • c、cpp、go

  • 解释型

    • 翻译成中间代码，再由解释器对中间代码翻译，生成机器代码
    • python、perl、js

其中java是特别一点,

比如 java 语言同时兼有编译型和解释型特点。整个流程如下：   将源代码（.java 文件）编译生成字节码（.class 文件），再通过 JVM（java 虚拟机）运行生成机器指令，由机器运行机器码。注意，此处生成机器语言前的操作是解释型，每次运行都要重新解释。因此，此处表明 java 是解释型。   但是，部分 JVM（java 虚拟机）有一种 JIT（Just in time）机制，能够将部分已经解释翻译的常用机器指令保存。下次不需要解释，直接运行即可。此时 java 是编译型。

编译性语言不如解释性语言跨平台性好   编译性语言例如c语言：用c语言开发了程序后，需要通过编译器把程序编译成机器语言（即计算机识别的二进制文件，因为不同的操作系统计算机识别的二进制文件是不同的），所以c语言程序进行移植后，要重新编译。（如windows编译成ext文件，linux编译成erp文件）。

3. 按照数据类型

  • 动态类型，在代码运行阶段对数据类型进行确认。

    • js、python等

  • 静态类型，确定类型

    • cpp、java

4. 按照程序设计

  • 结构化编程语言

    • c

  • 函数式编程语言

    • lisp
    • 程序员无需对程序修改，程序就可以并发运行。程序运行期间，不会产生死锁现象。原因是通过函数式编程所得到的程序，在程序中不会出现某一数据被同时修改两次及以上的情况，同样的，两个不同的线程就更不用说了。由于函数式编程有这样的优点，导致了程序员完全不用花费精力去考虑增加某个线程带来的并发问题。

  • 逻辑式编程语言

    • prolog(programming of logic)
    • 不用于开发软件，仅用于解决逻辑问题。输入事实和和谷子额，就可以自动查询其中的逻辑关系，完成逻辑运算。

  • 面向对象语言

    • java

4. 其他分类

  • 编程语言

    • 标准的图灵完备语言
    • c、c++、java

  • 脚本语言

    • 是为了缩短编程语言的“编写、编译、链接、运行”等过程而创建的计算机编程语言。是一种用来解释某些东西的语言，又被称为扩建的语言
    • python、php、js

  • 标记语言

    • 文本以及文本相关的其他信息结合起来，展现出关于文档结构和数据处理细节的电脑文字编码。
    • xml
